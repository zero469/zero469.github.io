DHT（分布式哈希表）学习笔记

发表于 2019-09-04

1 基本知识

1.1 概念

分布式哈希表（distributed hash table，缩写DHT）是分布式计算系统中的一类，用来将一个键（key）的集合分散到所有在分布式系统中的节点。
传统的散列表主要是用于单机上的某个软件中
分布式散列表主要是用于分布式系统（此时，分布式系统的节点可以通俗理解为散列表中的 bucket）

1.2 主要问题

一个hash表对外一般会有两个接口：

由于在分布式系统中存在很多的节点，那么put(key, value)主要解决的问题怎么根据key来确定将数据放在哪一个节点上

在分布式系统里，节点的加入与退出是很常见的，怎么高效的处理因节点的加入和退出导致的数据迁移也是需要考虑的

2 实现

2.1 简单hash

假如有N个节点，那么确定一个key对应的数据需要存储到哪一个节点时，可以采用如下算法计算数据存放的位置
location = key % N
该算法在物理节点数量和数据存放位置之间有强耦合，即当N发生变化时，原数据的位置计算就会出错，需要重新计算位置并移动数据

有节点退出也是类似的

2.2 一致性hash

简单hash取模，是将数据分布在一条直线上，而一致性hash的思路很巧妙，把hash取模的那个值域空间首位相连，变成一个环：

对A B C D四个节点，根据其hash(id)算出的值将其映射到环上的对应位置，对于外部value，首先算出其hash(key)，然后根据算出的值将其映射到环上，其实际存放的位置则是从其环上位置顺时针方向遇到的第一个节点。

2.2.1 增减节点


增加了节点E，那么我么只需要重新计算原来存放在C上的数据的hash值，然后把B和E中间的部分迁移到E上，其他数据不变
减少节点也是同理

2.2.2 优缺点

  • 优点：增删节点时影响较小
  • 缺点：负载均衡很差，难以保证节点在环上的位置是均匀分布的，且发生数据迁移时，可能会导致某一个节点的负载过大

    由于节点数的增加，节点在环上的分布会更加的均衡，数据只需要存放在其第一个遇到的虚拟节点对应的物理节点上，这样负载均衡的问题得到了解决，节点退出或加入时导致的数据迁移也会更加的均衡

nm命令生成导出符号表中符号含义

发表于 2019-09-04

符号表

  • 符号表是一种用于语言翻译器（例如编译器和解释器）中的数据结构。在符号表中，程序源代码中的每个标识符都和它的声明或使用信息绑定在一起，比如其数据类型、作用域以及内存地址。
  • 符号表只记录静态存储的符号，如静态变量，全局变量等

全局变量和static变量

  • 不同变量的主要区别是存储位置和作用域的不同
  • 全局变量是定义在所有函数外部的变量，作用域是整个程序的所有代码文件
  • 局部变量是定义在函数内部的变量，作用域只在函数内，且存储在栈中，使用完后会被销毁
  • 用static修饰的变量称为静态变量，静态变量分为全局静态变量和局部静态变量
  • 全局静态变量的作用域为当前代码文件
  • 局部静态变量的作用域为当前函数，但是其在整个程序运行期间都存在，只是不能被使用
  • 所有的全局变量和静态变量都存在静态存储区，生存周期和程序的生存周期一样

查看c语言符号表

1. 将源代码编译为目标文件(*.o)

2. 使用 _nm_ 命令从目标文件中提取符号表，然后将输出结果重定向到 result.txt 中

符号表中常见符号类型总结

符号类型 说明
U 未定义符号，在当前文件中使用，但是定义在其他文件中
T/t 代码段中符号，一般是非static/static函数
B 符号处于BSS段内，一般是未显式初始化的全局变量
b 符号处于BSS段内，一般是未显式初始化的(全局/局部)静态变量
C 未初始化的全局变量，并且在别的地方被引用
D/d 初始化过的全局或静态变量

小写符号类型一般表示该符号是locall的，大写的一般表示该符号是global(external)的
